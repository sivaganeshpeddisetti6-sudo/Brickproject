Security & Privacy

Built for sensitive workflows from day one.

Privacy-first architecture

  • No training on sensitive user data
  • No cross-user memory
  • Per-agent isolation (each bot acts independently)
  • Retention controls (forget after task / time-based)

Security controls

  • Encrypted secrets vault for credentials/tokens
  • Least-privilege access scopes
  • Approval gates before critical actions
  • Audit logs for every request and change
Compliance note: If you operate in regulated industries (e.g., healthcare/education), you’ll align policies and controls to applicable laws and requirements (HIPAA/FERPA, etc.). Only claim certifications after you actually complete them.

FAQ

Quick answers to common questions.

Is this just a chatbot?

Yes.It can answer your questions, and also AgentFlow is designed to securely complete tasks across systems, with approvals and audit logs.

How do you handle passwords and sensitive data?

Use encrypted storage for secrets and prefer token-based access. Agents only get the minimum access needed to complete a task.

Can anyone publish modules?

Yes, but community modules should run in a sandbox with permission scopes, reviews, and versioning so users stay safe.

Do you train AI models on user data?

No training on sensitive user data. You can also configure retention controls to delete task data after completion.
